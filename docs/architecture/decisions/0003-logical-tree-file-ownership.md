# ADR-0003：逻辑树与真实文件分离

状态：Accepted
日期：2026-07-22

## 背景

用户需要文件树、跨项目组织、大型媒体引用和虚拟文件系统。物理路径会变化，重复移动大文件成本高，删除语义也容易误伤用户原文件。

## 决策

数据库中的 Node 构成逻辑树。Asset 通过 Managed 或 Linked locator 指向物理文件。Node、Asset 与物理路径各有独立身份和生命周期。首版不使用 File Provider。

## 后果

- 一个 Asset 可在多个逻辑位置出现而不复制文件。
- 逻辑重命名和移动不会意外修改原文件。
- UI 必须清楚区分“移除入口”“移到废纸篓”和“删除原文件”。
- Linked 文件需要 Bookmark 失效、卷离线和重新授权流程。

## 未选择

- 直接渲染 Finder 目录：无法表达多引用、Smart Collection 和稳定身份。
- 所有文件强制复制：大型视频和既有素材库的成本不可接受。
- 首版 File Provider：它解决远端同步副本对 Finder 的暴露，不是应用内逻辑组织的必要条件。

## 复审触发条件

产品需要把 Verso 的远端托管文件按需同步并显示在 Finder Sidebar 时，另建 File Provider ADR。
