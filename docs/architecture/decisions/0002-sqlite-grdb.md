# ADR-0002：SQLite/GRDB 作为业务事实来源

状态：Accepted
日期：2026-07-22

## 背景

产品需要可控迁移、复杂树与引用查询、全文搜索、审计、后台任务和未来同步兼容。大文件不能存进关系数据库。

## 决策

SQLite 保存业务事实与元数据，通过 GRDB 访问；开启 foreign keys 和 WAL。原始多媒体与 Markdown 内容存文件系统。Repository/Port 隔离 GRDB 类型，UI 与 Domain 不接触数据库对象。

## 后果

- schema、索引、事务、迁移和备份行为明确且可测试。
- FTS5 可与业务事务紧密配合。
- 团队需要维护 SQL 和 migration fixture。
- GRDB 是第三方依赖，版本升级必须通过依赖审计和完整迁移/并发测试。

## 未选择

- SwiftData 作为主存储：适合 Apple 生态对象持久化，但当前产品更重视显式 SQL、FTS、迁移和跨版本诊断能力。可用于不关键的系统集成功能，但不得形成第二个业务事实来源。
- 把全部数据存为 Markdown/JSON：难以保证关系一致性、查询性能、事务和迁移。
- 独立数据库服务：破坏离线和单机部署的简洁性。

## 复审触发条件

SQLite 已被真实测量证明无法承担目标数据量，或多人实时协作协议要求完全不同的服务端事实模型时复审。
